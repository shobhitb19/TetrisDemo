
# Phase 4 – Polish, Accessibility & Trust

## Goal
Make the game usable, inclusive, and professional.

## Accessibility
- Keyboard-only support
- Clear focus states
- High-contrast colour palette
- Screen-reader friendly labels

## UX Enhancements
- Score tracking
- Level progression
- Speed increase over time
- Clear game-over messaging

## Quality Standards
- Well-commented code
- Clear separation of concerns
- Easy to modify for demos

## Success Criteria
- Works for a broad audience
- Feels intentional and trustworthy
- Comfortable to show externally
